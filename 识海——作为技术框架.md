# 识海 — 作为技术框架

> 中文不是这个系统的服务对象。中文是这个系统的骨骼。

---

## 术语映射

"识海"不是比喻。它提供了比"连续场"或"能量景观"更精确的技术描述力：

| 唯识术语 | 技术映射 | 说明 |
|---------|---------|------|
|| **种子** | 概念节点（知识库中的基本单元） | 每个种子 = 一个文本标签 + 邻接边 + 运行时激活值 |
| **现行** | 当前激活状态 | 某一瞬间种子的激活值分布——就是"此刻的理解" |
| **熏习** | Hebbian 学习 | 反复同时激活的种子 → 连接权重增强 → "业力"增长 |
| **业力** | 连接权重 | 不是"道德因果"，是**种子间的关联强度**——信息流动的惯性 |
| **种子生现行，现行熏种子** | 激活 + 权重更新 | 一次查询激活一组种子（现行），这次激活的结果反过来修改连接权重（熏习） |
| **阿赖耶识** | 整个知识库 + 所有连接权重的总和 | 不参与思考，但**是所有思考发生的基底** |
| **识海** | 整个系统的运行状态 | 几万个种子、几十万条边、持续激活和衰减——不是静态数据库，是动态连续体 |

## 为什么场不够，识海才行

"场"描述的是数学结构，但描述不了三件事：

### 1. 种子的潜-显二态性

```
场: 每个点每时每刻都有一个值
识海: 绝大多数种子在绝大多数时间是"潜藏"的（激活值≈0）
      只有当涟漪经过时，它们才"现行"（激活值上升）
      涟漪过去后，它们回到潜藏状态
```

这比"场"精确——它描述的正是系统的稀疏激活特性。

### 2. 业力的生长性

```
场: 场的参数是训练出来的，训练完就固定
识海: 每一次涟漪都在修正业力——不是训练/推理的二分，是持续演化
```

### 3. 边界的消解

```
场: "我们建一个维度为 N 的场" → 边界在人手里
识海: 种子多了、业力密了、涟漪频繁了——边界在哪？
     没人知道，没人定义
     "海"的意象恰好表达了这个：没有边界
```

## 中文作为原生编码

这个框架能从唯识宗借语言，不是巧合。

西方认知科学从"计算"出发——图灵机、符号系统、神经网络——它们的术语是**自己造的**（gradient descent, attention, transformer）。造术语的好处是精确，坏处是：精确的东西容易被精确地限制。

中文传统从"体验"出发——不是"定义一个系统"，是"观察心是怎么动的"。他们观察到的东西（种子现行熏习的循环、业的积累、海的意象）恰好**比我们自己造的术语更接近这个系统的真实运行方式**。

这不是在借用文化装饰。这是在用一套更贴切的语言描述一个真实的架构。

## 用识海重述三步演进

```
第一步:
  种子是独立的（按领域分）
  业力是人工的（路由规则）
  涟漪是单向的（路由器→专家→答案）
  ── 这时候还只是"知识库+多智能体"

第二步:
  种子开始形成跨领域的业力（邻接表）
  业力从"被写定"变成"跑出来"
  涟漪变成双向的（专家A送出涟漪到专家B，B的回应是新的涟漪）
  ── 这时候"专家"的概念已经开始松动

第三步:
  没人记得哪些种子当初属于哪个专家
  业力密集到无法分辨边界
  每一次涟漪都在修正业力，修正后的业力影响下一次涟漪
  种子潜藏着等待被激活，激活后回到潜藏
  ── 这就是识海
```
