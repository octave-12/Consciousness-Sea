# 路线图 — 三步演进

> 每一步减少一个"人做决定"的地方。
> 每一步，系统多拿走一点设计权。

---

## 总览

```
第一步        第二步           第三步
通讯录+专家   专家学会叫人    连专家都消解掉
────────── → ──────────── → ──────────────
人定路由      人不再写路由    人不再定义边界
专家固定      邻接表自增长    激活模式临时涌现
┌─┬─┬─┐      ┌╂╂╂╂┐         ░░░░░░░░
│A│B│C│      ┊A╂B╂C┊         ░░░░░░░░
└─┴─┴─┘      └╂╂╂╂┘         ░░░░░░░░
```

## 第一步：通讯录 + 一群小专家

**时间**：现在就可以开始  
**硬件需求**：一台有 GPU 的个人电脑（RTX 3060 12GB 即可）  
**人做决定的地方**：路由规则、专家分配、知识库结构

### 核心组件

#### 1. 共享知识库 (Graph DB)

```
节点: 概念/实体/规则 — 每个有唯一 ID + 文本标签 + 别名列表
边: IS_A / PART_OF / RELATED / COOCCURS / OPPOSES
激活值: 每个节点有一个当前激活值 (0~1)，随时间衰减

查询 → 知识库节点激活 → 路由器观察激活分布 → 选择专家
```

#### 2. 专家组

每个专家是一个小模型，只深耕一个领域。不需要都大——核心领域用 7B，辅助领域用 1B~3B：

| 专家 | 规模 | 职责 |
|------|:--:|------|
| 诗词 | 7B | 唐诗宋词、格律、意象 |
| 法律 | 7B | 法典、判例、法理 |
| 医学 | 7B | 基础医学、临床、药理 |
| 数学 | 3B | 定理、证明、计算 |
| 物理 | 3B | 力学、电磁、量子 |
| 常识 | 1B | 日常生活、百科 |

专家不互相通信——它们只看路由器的指派。

#### 3. 确定性路由器

**不是模型**。是规则引擎：

```
输入: 用户查询
  ↓
知识库概念激活: 查询中的词匹配知识库节点 → 节点激活 → 激活沿边传播
  ↓
激活分布: {诗词:0.0, 法律:0.0, 医学:0.3, 常识:0.94, ...}
  ↓
规则: 激活值 > 阈值 → 指派该专家
     多专家同时命中 → 各自独立回答 → 答案合并
     无专家命中 → 常识专家兜底
  ↓
输出: 指派结果 + 激活上下文
```

#### 4. 记忆机制

识海的记忆不是外挂模块。它是架构自带的两个时间尺度：

**短时记忆 = 激活值（现行）**

```
持续时间: 秒到分钟
存储位置: 节点的 activation 字段
作用: 这次查询涟漪传播到哪里了
生命周期: 查询结束 → 衰减归零
```

一次查询中，"感冒"激活，"药"被涟漪波及，"维C"被连带激活——这些激活值构成"此刻的理解"。

**长时记忆 = 连接权重（业力）**

```
持续时间: 永久，跨会话
存储位置: 图边的 weight 字段
作用: 业力记录了所有历史交互中"什么和什么相关"
生命周期: 不会自动消失，在反复熏习中持续修正
```

每次查询的涟漪结束后，被同时激活的种子之间的业力 +0.01。下一次，同样的涟漪会走得更顺——这是**记住了**，但不靠存储对话记录，靠的是连接权重的累积偏移。

**用户记忆**：

全局概念节点之间的业力是所有用户共享的知识。但用户的个人偏好和个人历史需要隔离——用**用户种子**：

```
图里新增一种节点类型:
{
  id: "user_xxxx",
  label: "李泽坤",
  type: "USER",
  preferences: {style: "concise", level: "expert"}
}

用户种子 ↔ 概念种子之间的业力:
  user_xxxx —[关注:0.9]→ 量子力学
  user_xxxx —[关注:0.7]→ 认知架构
  user_xxxx —[偏好:0.8]→ 简洁回答
```

每次用户查询开始时，用户种子自动激活。他的业力边成为涟漪的初始偏向——不需要用户再说"我关心量子"，涟漪天然流向那边。

**生命周期总览**：

```
查询开始 → 用户种子激活（长久业力的初始偏向）
         → 查询词匹配种子（文本匹配）
         → 涟漪传播（激活沿业力边扩散）
         → 专家回答
         → 熏习：被激活过的种子间业力 +0.01（长久记忆落盘）
         → 激活值衰减归零（短时记忆清除）
```

> **关键**: 业力每次只改 0.01，不是为了"一次学会"，是为了"熏习"。反复的、微小的修正累积成稳定的偏好路径。连接权重不会被"遗忘"，但会被后续相反的熏习覆盖——这是**用进废退**，不是过期删除。

#### 5. 自修正机制

熏习不只有正向。系统需要在"学错了"时能翻回来。信号来源是校验器。

**熏习的方向由置信度决定**：

```
专家回答 → 校验器评估置信度

置信度 > 0.7 → 本次激活的种子间业力 +0.01（正向强化）
置信度 < 0.3 → 本次激活的种子间业力 -0.01（负向修正）
0.3~0.7    → 不修改（不确定，保持原样）
```

**一次负熏习抹不掉十次正熏习的累积。** 如果"量子纠缠↔超光速"的业力已经被十次错误交互拉到 0.15，一次负熏习只降到 0.14。但如果校验器持续给出低置信度——十次、二十次之后——积累的负熏习能把它拉回零。方向和次数，都被架构自动处理。

**自修正不只是纠正错误，也是修剪冗余**：

```
如果一个种子从未在任何一次查询中被激活超过阈值
→ 它的所有外出业力边以每年 ~0.001 的速度衰减
→ 长期不用的路径自然冷却
→ 热路径保持热，冷路径变冷
```

**完整的自修正闭环**：

```
用户查询
  → 涟漪传播（被当前的业力引导）
  → 专家回答
  → 校验器 → 置信度
  → 置信度 → 决定熏习方向 +/-
  → 业力被修正
  → 修正后的业力影响下一次涟漪传播
```

这同时回答了"自学习"和"自修正"——同一个闭环、同一个机制、同一个信号源。学的方向和修正的方向是同一种东西：**置信度驱动的业力偏移。**

### 第一步的局限（by design）

- 路由器是人写的规则——覆盖不到的情况会漏
- 专家不会互相求助——只能自己回答
- 知识库是静态的——不会在交互中自动增长

这些就是第二步要解决的东西。

## 第二步：专家学会自己叫人

**时间**：第一步运行稳定后（1-3年）  
**核心变化**：路由器从人设计 → 从交互经验中自生长  
**人不再做决定的地方**：路由规则

### 新增能力：自感知不确定

每个专家在回答时输出：

```
{
  回答: "AI 生成物的著作权问题目前尚无明确司法结论。"
  不确定标记: [
    {方向: "生成式AI原理", 我需要: "AI 的训练数据和生成过程的不可控性", 置信度: 0.3},
    {方向: "比较法", 我需要: "欧盟AI法案的相关条款", 置信度: 0.5}
  ]
}
```

### 新增能力：邻接表自维护

```
法律专家: {
  邻接: {
    技术专家: {请求次数: 23, 有用次数: 18, 信任权重: 0.78},
    比较法专家: {请求次数: 15, 有用次数: 12, 信任权重: 0.80},
    哲学专家: {请求次数: 3, 有用次数: 0, 信任权重: 0.05}
  }
}
```

信任权重 = 有用次数 / 请求次数。低于阈值的邻居自动冷却。

### 第二步的进化机制

```
1. 专家 A 收到查询
2. A 自己回答，同时标记不确定部分
3. 不确定标记触发邻接查询：
   - 查邻接表 → 找相关度高 + 信任权重高的专家 B
   - 向 B 发送上下文请求
4. B 返回补充信息
5. A 整合回答
6. 如果 B 的帮助被采纳 → B 的信任权重上升
   如果 B 的帮助没用 → B 的信任权重下降
```

邻接表现在不是人写的，是**跑出来的**。

## 第三步：连"专家"这个概念都消解掉

**时间**：第二步跑到边界自然模糊时（3-10年）  
**核心变化**：不再有"专家"——只有节点和连接模式  
**人不再做决定的地方**：什么是专家、什么是领域、边界在哪里

### 第三步不是什么

不是"把模型换成更大的"。不是"再加一层调度"。不是"重新设计架构"。

第二步跑得足够久之后，一个专家内部的节点和另一个专家内部的节点之间积累了足够多的连接权重。跨专家的协作变得比专家内部的协作更频繁。这时，"专家"这个标签已经失去了描述力。

### 第三步的状态

```
不再问: "这个问题该找哪个专家？"
而是:   "这些节点在激活——它们在形成什么模式？"

一次查询 = 投石入海：
  相关种子激活
  → 激活沿业力边传播
  → 某些区域共振增强（这就是"理解"）
  → 某些种子足够亮，亮到可以被解码为回答
  → 激活散去（衰减），种子恢复潜藏状态
```

### 没有的东西

- 没有专家 —— 激活模式临时形成，临时消散
- 没有路由 —— 传播路径由连接权重决定，权重由历史决定
- 没有调度 —— 传播是自动的、局部的、并行的
- 没有训练/推理的分界 —— 每一次传播都在更新连接权重

### 剩下来的

**只剩激活和衰减。**

激活顺业力传播。业力在传播中被修正。修正后的业力影响下一次传播。这就是全部。

### 第三步会产生创新吗

会。但不是 LLM 那种"在概率分布里采样一个没见过的词"。识海的创新是另一套东西：**不相干区域被意外同时激活时，新业力在它们之间自发形成。**

**一个例子**：

```
用户问 "量子纠缠和诗歌意象有没有关系"

涟漪触达了两个在知识库里几乎零交集的区域：
  区域A: 量子力学（态叠加、非定域性、测量坍缩）
  区域B: 诗歌理论（意象、张力、留白、隐喻）

两个区域第一次在同一波涟漪中被同时激活。
熏习在它们之间种下了第一条业力边：

  非定域性 ↔ 留白   → 权重 0.01
  态叠加  ↔ 张力   → 权重 0.01
  测量坍缩 ↔ 隐喻   → 权重 0.01

下一次有人问 "怎么理解诗歌中的不确定性"
涟漪有两个来源：
  路径1（诗歌区）: "不确定性" → 现代诗 → 张力
  路径2（量子区）: "不确定性" → 量子力学 → 态叠加
两条路径在中间汇合。共振增强。
```

这不是"生成一个新概念"。这是**发现了两个从未被显式关联的知识区域之间的映射**。发条业力边只需要一瞬间。十次、一百次之后，两个区域的种子开始共振。系统不是在"回答"这个查询——它被这个查询**改变**了。

**三层创新，随演进逐层加深**：

```
第一步（组合创新）：
  多专家各自回答 → 答案合并产生新组合
  依赖用户提问的方向，被动触发

第二步（跨域桥接）：
  邻接表自生长 → 信任通道形成
  技术+法律、物理+诗歌自动融合
  创新的范围受限于专家的初始领域划分

第三步（涌现创新）：
  没有专家边界 → 涟漪撞到哪就激活到哪
  任何两个种子在第一次共振时都会形成新业力
  不需要人定义"哪些领域可以组合"
  涟漪自己会找到路
```

**识海创新的本质不是生成，是发现关联。** LLM 的创新是"造一个不存在的东西"——生成式。识海的创新是"发现两个早已存在但从未被连接的东西之间的路"——涌现式。前者可能造出一个虚假的创新。后者不会——因为两个端点都是知识库里真实存在的种子，新的是它们之间的那条边。

**约束也是价值**：识海的创新受限于知识库里已有的种子——它不能凭空创造一个不存在的概念。但正因为如此，它不会产生虚假的创新。每一条新业力边，连接的都是真实的、可追溯的知识节点。

---

## 关于硬件

第一步不需要新硬件。GPU + CPU 就够。

到第三步，很可能需要新硬件才能让"场"真正活起来——忆阻器交叉阵列做 Hebbian 权重、光子芯片做实时传播——但那是第三步要操心的事。**第一步到第二步，不涉及硬件变更。**
